# Hupository — 박세종의 공개 인간 데이터 저장소

> 교육자에서 개발자로 전환한 1인 창업자 박세종의 public human repository.
> 사람은 읽기 쉽게, AI는 추론하기 쉽게 정리한 공개 자기소개/상태 저장소입니다.

## 30초 소개

- 학생들의 공부와 진로를 오래 도왔고, 지금은 교육과 기술을 연결해 사람들이 자기 삶을 더 잘 설계하게 돕는 서비스를 만들고 있습니다.
- 현재의 핵심 축은 `세종클래스`, `Hupository`, `PersonaMirror`입니다.
- 결과물만 보여주는 포트폴리오보다, 방향과 판단, 현재 상태까지 함께 공개하는 방식을 더 중요하게 생각합니다.

조금 더 압축하면 이런 사람입니다.

> 학생들의 취업과 성장을 오래 도왔고,
> 지금은 교육자 감각과 개발자 실행력을 함께 가져가는 빌더입니다.

## 지금 집중하는 것

2026-03-25 기준, 현재 방향은 아래 세 축으로 정리됩니다.

- `세종클래스`
  수학·물리·코딩·라이프디자인을 연결하는 1인 교육 플랫폼을 직접 기획·개발·운영하고 있습니다.
- `Hupository / PersonaMirror`
  한 사람의 데이터와 현재 상태를 구조화하고, 그것을 AI와 연결해 자기이해·협업·팀 핏 탐색에 쓰는 방향으로 발전시키고 있습니다.
- `AI 빌더 역량 고도화`
  `React 19`, `FastAPI`, `LangGraph`, `OpenAI API`, `PostgreSQL`, `Redis`를 중심으로 실제 서비스와 데모를 빠르게 만들고 검증하고 있습니다.

최근에는 특히 `긴 서비스 설명`보다 `이 사람이 누구고 왜 같이 일하면 좋은가`가 먼저 읽히는 front door를 만드는 데 집중하고 있습니다.
즉, 이 저장소는 단순한 소개 문서가 아니라 현재의 문제의식, 실행력, 장기 방향을 함께 보여주는 공개 작업 공간에 가깝습니다.

## 협업 관점에서 보면

이 저장소를 처음 보는 협업자라면, 아래 세 가지를 보면 박세종의 결을 빠르게 파악할 수 있습니다.

- 문제를 보면 설명으로 끝내기보다 직접 구조화하고 프로토타입으로 옮기는 사람인지
- 교육, 커뮤니티, 사용자 맥락 이해를 제품과 문서로 연결할 수 있는 사람인지
- 단기 기회에 흔들리기보다 장기 방향 안에서 현재 선택을 정리하는 사람인지

현재 강점은 대체로 이쪽에 있습니다.

- 사용자/문제 맥락 이해
- 문서화와 구조화
- 교육적 설명력
- 빠른 프로토타이핑과 end-to-end 실행

그래서 `AI/백엔드/인프라 깊이` 또는 `프론트엔드/UX 완성도`를 강하게 가져가는 동료와 만났을 때 특히 시너지가 큽니다.

## 왜 이런 저장소를 공개하나

대부분의 자기소개는 결과만 보여줍니다.
하지만 실제 사람은 결과보다도 방향, 맥락, 변화 과정으로 더 잘 이해된다고 생각합니다.

그래서 Hupository는 단순한 포트폴리오보다 아래 질문에 답하는 저장소를 지향합니다.

- 이 사람은 지금 무엇을 만들고 있는가
- 왜 그 방향을 선택했는가
- 어떤 경험과 기술 위에서 움직이는가
- 장기적으로는 어디를 향하고 있는가

## 이 저장소는 어떻게 작동하나

Hupository는 정적인 소개 페이지가 아니라, 계속 업데이트되는 개인 데이터 시스템입니다.

```text
정체성 / 경험 / 스킬 / 목표 / 현재 상태
                    ↓
             일일 기록과 주간 점검
                    ↓
      더 나은 선택과 더 선명한 자기이해
```

AI는 이 데이터를 바탕으로 생각을 정리하고, 장기 목표와 현재 행동의 정렬을 점검하는 데 쓰입니다.
하지만 출발점은 어디까지나 사람입니다.
먼저 사람이 읽고 이해할 수 있어야 하고, 그 다음에 AI가 잘 읽을 수 있어야 합니다.

## 구조를 간단히 보면

```text
hupository/
├── README.md              ← 사람용 front door
├── data/
│   ├── 01_identity/       ← 가치관, 자기인식
│   ├── 02_experience/     ← 경력, 프로젝트
│   ├── 03_skills/         ← 기술/소프트 스킬
│   ├── 04_story/          ← 삶의 서사
│   ├── 05_interests/      ← 관심사, 독서
│   ├── 06_goals/          ← 장기 목표 트리
│   ├── 07_matching/       ← 매칭 신호
│   ├── 08_now/            ← 현재 상태와 결정
│   └── 09_journal/        ← 일일/주간 리플렉션
└── ai/
    ├── index.yaml
    ├── context/
    ├── prompts/
    └── outputs/
```

더 자세한 구조 규칙은 [data/schema.yaml](data/schema.yaml)에서 볼 수 있습니다.

## 공개 원칙

- 개인식별정보와 민감정보는 공개 대상에서 분리합니다.
- 대신 삶의 방향, 실패, 전환, 현재 고민처럼 맥락을 이해하는 데 중요한 정보는 가능한 한 숨기지 않습니다.
- 좋은 모습만 전시하는 포트폴리오보다, 실제로 이해 가능한 기록을 더 중요하게 생각합니다.
