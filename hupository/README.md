# ✅ Hupository — 박세종 데이터 저장소

> **Hupository = (Human + Repository)** — 한 사람의 삶을 코드처럼 버전 관리하는 개인 데이터 인프라

2007년~2026.02약 19년간의 데이터를 누적하여 AI 서비스가 소비할 수 있는 형태와 사람이 읽을 수 있는 형태 \
모두를 동시에 지원하기 위해 고안됨 곧 이 데이터에 질문 할 수 있는 "휴포짓" 서비스 런칭 예정.

```
한 사람의 삶  →  구조화된 데이터  →  AI · 서비스 · 사람이 함께 읽는 저장소
```

## ✅ 저장소 구조 (Private Hupository)

```
hupository/
│
├── README.md                   ← 프로젝트 전체 구조
│
├── private/                    ← RAW 데이터 레이어 (19년 원본 아카이브)
│   ├── 00_profile/             ← 기본 정보 · 연락처 · 현재 스냅샷
│   ├── 01_identity/            ← 핵심 정체성 · 가치관 · 정체성 변화 이력
│   ├── 02_timeline/            ← 2007–2025 연도별 회고 (월별 하이라이트 포함)
│   ├── 04_work_learning/       ← 경력 · 프로젝트 · 기술 스택 · 학력
│   ├── 05_interests/           ← 관심 분야 (뇌과학·물리학·교육·SW·커뮤니티 등)
│   ├── 07_goals/               ← 단기/중기/장기 목표
│   ├── 08_matching/            ← 커뮤니티·역할 매칭 신호
│   ├── 09_privacy/             ← 공개·비공개·비밀 가시성 규칙
│   ├── 10_artifacts/           ← 블로그 원문 · 타임라인 이미지 · 이력서 스냅샷
│   └── 11_derived/             ← AI가 추론한 태그 · 프로필 요약
│
└── public/                     ← 정제된 데이터 레이어 (AI 서비스 최적화)
    ├── index.yaml              ← AI 진입점: 1분 요약 프로필
    ├── schema.yaml             ← 서비스 소비용 데이터 스키마
    ├── 01_identity/            ← 정체성 · 가치관 · 페르소나 (LLM 컨텍스트용)
    ├── 02_experience/          ← 경력 · 학력 · 프로젝트 상세 (매칭용 구조화)
    ├── 03_skills/              ← 기술/소프트 스킬 · 숙련도 레벨
    ├── 04_story/               ← 6챕터 서사 마크다운 (LLM 컨텍스트 주입용)
    ├── 05_interests/           ← 관심 도메인 · 깊이 · 트렌드
    ├── 06_goals/               ← 단기/장기 목표 (역할 유형 정렬)
    ├── 07_matching/            ← 매칭 벡터 · 신호 · 프라이버시 설정
    └── 08_ai_layer/            ← AI 생성 프롬프트 · 출력물 (자기소개서·이력서·태그)
```
